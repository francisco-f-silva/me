Software for Shop-Floor Management (Industrial Units)

Project Type: Command-Line Interface (CLI)
Context: BSc in Software Engineering
Date: March - June 2020

Developed, in a team of 4 members, a software system able to (i) receive the diversity of messages generated by the machines/equipments of an industrial unit; and (ii) process that data in order to extract useful information concerning production orders and their execution, working time of the machines, consumed quantities of raw materials, and final quantities of the intended products.

This project was done as part of the 4th semester of the BSc in Software Engineering. It was meant to integrate knowledge on the following subjects: Application Engineering, Languages and Programming, Computing Systems, and Computer Networks.

Please see below for more details on the architecture of the system.

Bitbucket
Technologies: Java | C | H2 Database

Architecture of the system

The user interacts with the Console Application. The Console Application accesses a common database to obtain information requested by the user. It is also able to communicate with different services (MCS, MPS) to trigger some actions (e.g. processing messages from the machines). These services are the ones interacting with the actual industrial machines, whose action was in our case simulated by a simple application.
